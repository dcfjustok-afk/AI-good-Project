## 1. MVP 目标

围绕“发现 + 理解 + 收藏”三件事，先完成以下闭环：
1. 本地拉取 GitHub AI 开源项目数据
2. 将项目数据和 AI 摘要持久化到 SQLite
3. 在桌面端完成列表、详情、筛选、收藏展示
4. 提供手动同步与失败回退能力

---

## 2. 交付范围

### In Scope
- GitHub AI 项目抓取
- 本地 SQLite 存储
- AI 摘要生成
- 首页榜单页
- 项目详情页
- 收藏页
- 基础筛选与排序
- 手动同步数据
- 错误重试与缓存兜底

### Out of Scope
- 多用户体系
- 团队评论协作
- 消息推送
- 趋势图表
- 多数据源聚合

---

## 3. 开发原则

- 先打通数据闭环，再补体验优化
- 先做可运行最小链路，再补高级筛选与评分
- Rust 负责数据采集、清洗、持久化、命令暴露
- React 负责展示、交互、筛选状态、缓存刷新
- 所有复杂能力都保留兜底方案，保证 MVP 可用

## 7. 风险与应对

### GitHub API 限流
- 使用 Token
- 控制请求频率
- 优先增量同步
- 本地缓存兜底

### AI 摘要质量不稳定
- 固定结构化输出模板
- 增加失败回退描述
- 先做摘要，不做复杂推理

### SQLite 与并发写入问题
- 同步任务串行执行
- 命令层限制重复触发
- 必要时引入连接管理

### 开发范围膨胀
- 严格围绕“发现 + 理解 + 收藏”
- 协作、推送、图表全部后置

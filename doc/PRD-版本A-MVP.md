# AI 开源项目情报站 PRD（版本 A：MVP 轻量版）

## 1. 文档信息
- 产品名称：AI 开源项目情报站
- 文档版本：V1.0
- 产品阶段：MVP
- 目标用户：部门内前端工程师、技术负责人、对 AI 开源趋势感兴趣的产品/设计同学
- 文档日期：2026-03-18

## 2. 产品背景
团队内部对 AI 开源项目的关注度越来越高，但信息主要分散在 GitHub Trending、社交媒体、社区文章和群聊中，存在以下问题：

1. 热门项目多，但噪音较大，难以快速判断是否值得关注。
2. 前端同学很难从 README 中快速识别“这个项目和我有什么关系”。
3. 缺少统一入口，团队无法共享和沉淀调研结果。
4. 收藏后的项目缺少持续跟踪，容易错过项目爆发或衰退节点。

因此，需要一个面向团队内部的 AI 开源项目情报平台，用于聚合、筛选、摘要和跟踪当前 GitHub 上优质的 AI 项目。

## 3. 产品目标
### 3.1 业务目标
- 建立团队统一的 AI 项目发现入口。
- 提升前端同学获取 AI 热门项目的效率。
- 降低项目调研成本，提升内部分享与试点效率。

### 3.2 用户目标
- 快速知道当前值得关注的 AI 项目有哪些。
- 快速理解项目定位、价值、技术栈和前端相关性。
- 根据兴趣方向进行筛选、收藏和持续跟踪。

### 3.3 MVP 成功标准
- 用户可以在 3 分钟内找到 3 个与自己工作方向相关的 AI 项目。
- 单个项目卡片的阅读时间控制在 30 秒以内即可理解核心价值。
- 首页项目列表可每日自动更新。

## 4. 用户画像
### 4.1 前端工程师
关注点：
- AI UI、Agent 前端、聊天交互、流式输出、RAG 前端应用
- 是否适合做 Demo、接入现有业务、复用交互设计

### 4.2 技术负责人
关注点：
- 当前热门方向
- 哪些项目值得团队调研或试点
- 哪些项目稳定、活跃、可持续

### 4.3 产品/设计同学
关注点：
- 项目能解决什么问题
- 是否有 Demo、截图、交互参考
- 是否值得做内部分享

## 5. 核心场景
### 场景 1：快速浏览热点
用户进入首页，查看今日/本周热门 AI 项目，了解最新趋势。

### 场景 2：按方向查找
用户按 Agent、RAG、AI Coding、多模态等方向筛选项目，找到与工作相关的仓库。

### 场景 3：快速理解项目
用户打开项目详情页，通过 AI 摘要、核心亮点、前端价值说明快速判断是否值得进一步查看。

### 场景 4：持续跟踪
用户收藏项目，后续查看项目的 star 增长、最近更新、热度变化。

## 6. 产品范围
## 6.1 MVP 范围内
1. GitHub AI 项目聚合
2. 热门项目列表与排序
3. 分类筛选
4. 项目详情页
5. AI 自动摘要
6. 收藏功能

## 7. 功能需求
### 7.1 首页热榜
#### 功能描述
展示当前热门 AI 项目列表，支持按时间与排序方式切换。

#### 子功能
- 时间维度：今日、本周、本月
- 排序方式：star 增长、fork 增长、最近更新、综合推荐
- 卡片字段：
  - 项目名称
  - 简介
  - star / fork
  - 最近更新时间
  - 语言
  - 分类标签
  - AI 摘要
  - 前端相关性标签

#### 价值
帮助用户快速发现值得关注的项目。

### 7.2 项目分类筛选
#### 功能描述
支持按 AI 方向和项目属性过滤列表。

#### 分类建议
- AI Coding
- AI Agent
- RAG
- LLM 应用
- 多模态
- 图像生成
- 视频生成
- 语音
- Prompt 工具
- AI UI / Frontend

#### 筛选项
- 编程语言
- 最近更新周期
- 是否有 Demo
- 是否适合前端关注
- Star 区间

### 7.3 项目详情页
#### 功能描述
展示单个项目的深度信息。

#### 详情字段
- GitHub 基础信息
- AI 摘要
- 核心能力
- 技术栈
- 适用场景
- 前端价值说明
- 项目链接
- 官网 / Demo
- License
- 最近更新情况

### 7.4 AI 摘要
#### 功能描述
对项目 README 和基础信息进行自动摘要。

#### 输出内容
- 一句话介绍
- 三点核心亮点
- 为什么值得关注
- 对前端有什么启发
- 学习成本：低 / 中 / 高

### 7.5 收藏功能
#### 功能描述
用户可将感兴趣的项目加入个人收藏列表。

#### 子功能
- 收藏 / 取消收藏
- 我的收藏页
- 按收藏时间排序

## 8. 非功能需求
### 8.1 性能
- 首页首屏时间小于 2 秒。
- 项目列表支持分页或无限滚动。

### 8.2 可用性
- 信息结构清晰，卡片化展示。
- 支持中文优先的信息呈现。

### 8.3 稳定性
- GitHub 数据拉取失败时展示缓存数据。
- 摘要生成失败时回退到原始描述。

### 8.4 安全性
- 内部系统登录后访问。
- GitHub Token 仅保存在服务端。

## 10. 推荐机制（MVP）
综合推荐分建议由以下维度组成：

- star 增长权重 40%
- 最近活跃度权重 30%
- 社区热度权重 20%
- 前端相关性权重 10%

输出为榜单默认排序依据。

## 14. 风险与应对
### 风险 1：GitHub API 限流
应对：服务端缓存、定时任务预抓取、使用 Token。

### 风险 2：AI 摘要质量不稳定
应对：设计摘要模板、增加结构化提示词、增加兜底策略。

### 风险 3：分类准确率不足
应对：先规则分类，后续引入 AI 自动打标。

## 15. MVP 结论
该版本聚焦“发现 + 理解 + 收藏”三件事，以最低成本建立团队内部 AI 项目情报入口，验证真实使用频率和筛选价值。

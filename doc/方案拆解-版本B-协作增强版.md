## 3. 技术方案

## 3.1 架构建议
增强版建议采用分层架构：

1. 采集层：抓取 GitHub 及后续数据源
2. 处理层：清洗、分类、打分、摘要
3. 存储层：结构化存储 + 榜单快照
4. 应用层：前端展示、订阅与协作
5. 推送层：周报与通知

## 3.2 推荐技术栈
### 客户端
- React + TypeScript
- Vite
- 路由：React Router
- 状态：Zustand
- 服务端状态：TanStack Query
- 图表：ECharts 或 Recharts
- UI：Tailwind CSS + 组件库

### 桌面端 / 容器
- Tauri 作为桌面容器

### 服务端
可选两种路径：

#### 路径 A：继续基于 Tauri Rust 扩展
适合纯内部桌面工具、低部署成本。

#### 路径 B：拆出独立后端服务
- Node.js / NestJS 或 Rust Web 框架
- 更适合后续多人协作、消息推送、权限体系

如果产品未来会给整个部门长期使用，建议优先考虑独立后端。

## 3.4 数据处理模块
### 模块 1：项目采集器
负责抓取仓库基础信息、统计指标、README、topic。

### 模块 2：标签分类器
基于关键词规则 + 模型分类输出：
- Agent
- RAG
- AI Coding
- 多模态
- AI Frontend

### 模块 3：评分引擎
综合以下指标：
- star 增长
- 最近更新
- issue / PR 活跃度
- 社区讨论度
- 团队相关性

### 模块 4：摘要引擎
输出结构化中文摘要、前端价值说明、风险说明。

### 模块 5：趋势分析器
基于每日快照计算趋势变化与榜单波动。

## 4. 与当前项目的结合建议
当前工程已经具备 Tauri + React 基础，可分两步走：

### 短期
先用当前桌面应用实现单机版浏览、筛选、摘要、收藏。

### 中期
抽离数据采集与处理服务，使多用户共享同一份项目数据与评论体系。

## 5. 总结
增强版方案更适合做成部门级内部平台。其核心不只是展示数据，而是把“热门项目发现”升级为“团队协作判断 + 趋势追踪 + 选型支持”。
